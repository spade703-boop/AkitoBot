# RPG 成长基线

这份文件是 RPG 与送礼联动链路的保底硬约束。

以后继续调签到、打怪、组队、世界 BOSS、送礼回流时，先看这里，再动数值。
如果实际配置和这份基线冲突，默认以这份基线为目标。

## 设计前提

- 核心用户不是最活跃的那批人，而是“手里有积分、暂时没人送礼、也不一定有人组队”的普通群友。
- RPG 必须是一条可独立成立的轻量成长线，而不是只给社交活跃玩家加速的附属系统。
- 送礼仍然是主社交线；RPG 可以挤占一部分积分消耗，但不能把送礼系统的存在感吃掉。
- 签到必须保留基本激励感；经验可以轻，但不应该轻到让人感觉“签了跟没签一样”。

## 单人保底合同

适用对象：

- 每天签到
- 主要打单人 `今日打怪`
- 不依赖组队
- 不依赖世界 BOSS
- 不把“有人送礼 / 有人陪玩”当作前提

目标线：

| 周期 | 单人保底等级 | 体感目标 |
|---|---:|---|
| 30 天 | Lv5-Lv6 | 明显变强，但仍在前中期 |
| 90 天 | Lv10-Lv13 | 进入稳定中段，开始接触更难怪 |
| 180 天 | Lv16-Lv20 | 接近成熟期，但不应完全毕业 |

约束说明：

- 单人玩家不能因为没人组队，就明显落后到“玩不下去”。
- 单人线允许慢，但不允许长期停滞。
- 高活跃社交玩家可以更快，但整体收益建议控制在单人线的 `1.2x-1.5x`。

## 怪物投放基线

怪物线按“半年内逐段推进”设计，不追求两个月内把主怪池全见完。

| 阶段 | 主体怪物 |
|---|---|
| 0-30 天 | 史莱姆 / 泥怪 / 哥布林 / 野狼 |
| 30-60 天 | 座狼 / 骸骨兵 / 食人魔 |
| 60-120 天 | 魔铠兵 / 石像鬼 |
| 120-180 天 | 双足飞龙 / 龙 |

落地要求：

- `龙` 不能在前中期就进入常规遭遇池。
- `双足飞龙` 应该属于明显后段的怪物，而不是一个月左右就频繁见到。
- 调整怪物节奏时，优先改 `combat.encounter_brackets`，不优先抬 `monsters[*].power_req`。

## 送礼经济基线

单人玩家也必须保有稳定参与送礼的能力。

建议目标：

- 单人玩家每周至少能稳定送出 `3-5` 次 `100-200` 档礼物。
- `270+` 档礼物应当有一定积攒感。
- `819 / 1112` 档礼物应保留明显仪式感，不应因 RPG 回流而变成日常消耗。

## 调参优先级

当成长线跑得过快时，按这个顺序处理：

1. 先压 `签到经验`
2. 再改 `怪物遭遇分段`
3. 还不够，再看 `打怪经验 / 等级曲线`
4. 最后才动 `组队 / 世界 BOSS` 奖励

原因：

- 签到应该是背景推进，不该成为主要升级来源。
- 怪物分段决定“这段时间在玩什么”，属于体验节奏。
- 打怪经验和等级曲线会直接影响长期成长度，改动更重。
- 组队和世界 BOSS 更适合做趣味与社交补偿，不适合作为单人线的补丁。

## 维护规则

- 以后如果 README、维护文档、配置注释和这里冲突，以这里为准。
- 每次做 RPG 大改前，先确认这 4 件事：
  - 单人 30 / 90 / 180 天等级是否还落在目标区间附近
  - 高阶怪是否仍然被压在后段
  - 单人玩家是否仍有送礼参与能力
  - 社交玩法是否只是“更有趣”，而不是“强者更强”
