# 版本更新规范

## 原则

项目分两区管理：

| 区域 | 路径 | 规则 |
|------|------|------|
| **版本控制区** | `/data` 以外所有文件 | 纳入 Git，随版本迭代更新 |
| **本地调试区** | `/data` 全部内容 | **不纳入版本控制**，仅保留在本地作调试样本 |

## 更新流程

1. 修改 `/data` 以外的代码或配置
2. 确认 `.env`、`.claude/settings.local.json` 未被追踪（`.gitignore` 已配置）
3. 若新增了可配置项，同步更新 `.env.example`
4. `git commit` + `git push` 推送到私有仓库

## 敏感信息遮蔽

- `.env` — **绝不提交**，含 DeepSeek / Tavily / 智谱 API Key
- `.claude/settings.local.json` — **绝不提交**，含本地 Auth Token
- `.env.example` — 可以提交，仅保留字段名和占位符
- 代码中不应硬编码任何密钥、Token、QQ 号（统一走 `.env` 读取）

## /data 目录说明

`/data` 目录下的所有文件均为本地调试样本，包括但不限于：

| 内容 | 说明 |
|------|------|
| 记忆文件、印象数据库 | 运行时自动生成 |
| 图片图库、派生头像 | 本地收集的素材 |
| 人设/提示词 JSON | 迭代中的调参数据 |

这些内容**不跟随版本发布**。如需迁移到新环境，手动拷贝 `/data` 目录即可。

## 版本号

采用 `主版本.次版本.修订号`（如 `0.2.1`）：

- **主版本**：架构大改、不兼容的 API 变更
- **次版本**：新功能模块、新指令
- **修订号**：Bug 修复、Prompt 调优、小改动

## Commit 规范

```
feat: 新增功能
fix: 修复 Bug
docs: 文档更新
refactor: 重构
chore: 杂项（依赖、配置等）
```
